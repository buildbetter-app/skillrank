# skillrank/web — build spec (FROZEN)

Build a **Next.js (App Router, TypeScript) static site** in `web/` that is the
skillrank registry — a retro-terminal (TUI) skill directory. Deployed as its own
Vercel project (Root Directory = `web/`) → skillrank.dev. Everything static/SSG.

## Design — port `web/design-reference.html` EXACTLY
`web/design-reference.html` is the approved, single-file visual design (amber
"PHOSPHOR" CRT aesthetic). Reproduce it faithfully in React/Next:
- Amber phosphor dark theme + greenbar-printout light theme, via the SAME CSS
  custom-property tokens + `@media (prefers-color-scheme)` + `:root[data-theme]`.
- CRT scanline/vignette/flicker overlays; block cursors; box-framed panes; sticky
  bottom keybar; sticky top brand bar. Keep the exact palette and type treatment.
- System monospace stack ONLY (no webfonts). Respect `prefers-reduced-motion`.
Move its inline CSS into a global stylesheet; move its inline JS into React
state/effects. Do not redesign — match it.

## Data — `web/data/catalog.json` (87 entries, real)
Regenerated by `web/data/build-catalog.mjs` from `web/data/sources.seed.json`
(`node web/data/build-catalog.mjs`). Load `catalog.json` at BUILD time (import or
fs read in a server component). Entry schema:
```ts
type Skill = {
  slug: string; display_name: string; category: string; tags: string[];
  source_type: "github"; source_url: string; source_repo: string; source_subpath: string;
  description: string;
  signals: { stars: number|null; installs: number|null };   // installs is null (no telemetry yet)
  eval: { success_delta_pct: number|null; token_delta_pct: number|null; n_trials: number|null; tier: "official"|"community"|"self" };
  score: number; provisional: boolean; added_at: string;
};
```

## HONESTY RULES (hard)
- `installs` is `null` everywhere → render as `—` or omit; never fabricate usage.
- `eval.*` deltas are `null` → render **"eval pending"**, never a number.
- `score` is **provisional** (star-derived, capped at 75) → badge it "provisional"
  and include a one-line explainer: *SkillRank score blends community stars, real
  usage, and our eval lift; provisional until a skill is evaluated — so popularity
  alone can't reach the top tier.* Stars are REAL and may be shown as-is.

## Pages
1. `/` — the registry (port of the reference):
   - **Install hero at top**: `curl -fsSL skillrank.dev | sh` (copy button) + a line
     for `skillrank setup` (ambient/MCP) and `skillrank recommend`.
   - **Sticky search** that live-filters by name/category/tags/description.
   - **Sidebar**: sort (score / stars / eval lift — lift sorts pending last) +
     category list with live counts (derived from catalog).
   - **Ranked leaderboard**: rank #, skill, category·tags, ★ stars, tier badge,
     Δ success ("pending"), provisional score bar. Rows expand inline to a detail
     (description, signal breakdown, `skillrank install <slug>` copy, source link,
     "view full page" → `/skill/[slug]`).
   - **Keyboard**: `/` focus search · `↑↓`/`j k` move selection · `Enter` expand ·
     `g` → GitHub · `i` → install hero. Sticky keybar shows these.
   - Theme toggle button (flips `data-theme` on `<html>`).
2. `/skill/[slug]` — static detail page per skill (`generateStaticParams` over catalog):
   - name, category, tags, description, source repo link (`source_url`), ★ stars,
     tier badge, provisional score + explainer, **eval pending** panel, and an
     install block (`skillrank install <slug>` + the curl one-liner). SEO `<title>`
     + meta description. Same PHOSPHOR styling. Back link to `/`.

## Tech / constraints
- Next.js latest, App Router, TypeScript, fully static (SSG). No runtime server, no
  DB, no external network at runtime. No external UI/font libraries — custom CSS only.
- Node 22. Use `npm`. Standard `next build` must pass clean; prefer `output: "export"`
  compatible (static). Reasonable `metadata` for SEO.
- **ONLY create/modify files under `web/`.** Do NOT touch `crates/`, `registry/`,
  `site/`, `.github/`, `Cargo.*`, or anything outside `web/`.
- Keep `web/data/*` and `web/design-reference.html` as-is (read them; the reference
  html may remain in-repo as a design doc).
